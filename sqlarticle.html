The database tracks professors giving lectures at universities. Some universities have more than one campus and professors can give more than one lecture. The database's name is ProfessorTracker The database has 5 tables: 1. Professors 2. TopicOfLecture 3. LecturesDone 4. Universities 5. Locations ```sql -- The database and table names do not have -- spaces in between the words. -- Database and tables use CapitalCase. -- The database is labled dbo.ProfesorTracker. -- The the tables are labeled as such dbo.Professors dbo.TopicOfLecture dbo.LecturesDone dbo.Universities dbo.Locations ``` The 5 tables are **interrelated** to each other. That is why _Structured Query Language (SQL)_ are known as **relational databases**. * Every table will have a primary key :key: that will be unique to that table. * Every table will have ``CreateDate`` and ``LastUpdated`` columns. * The tables will have surrogate keys (ids) that will align to primary keys (ids) to create table relationships. * A University might have multiple locations. Example: _Pepperdine_ has 2 campuses. One university, 2 locations. This is a **One to many relationship**.
